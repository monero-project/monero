# I2P SAM support in Monero

`monerod` supports I2P's [SAM](https://i2p.net/en/docs/api/samv3/) protocol (version 3.1) for automatically configuring and managing I2P connections. It is also possible to use I2P by means of SOCKS proxies, however this requires extra setup and can [leak metadata](https://i2p.net/en/docs/api/socks/); instead, SAM is the recommended way to use Monero over I2P.

When passed to the daemon, the `--i2p-sam` argument will effectively override the `--tx-proxy` and `--anonymous-inbound` parameters for I2P to use SAM. It is worth noting that the blockchain cannot be downloaded over anonymity networks, including I2P.

## Protocol parameters

| Parameter        | Value                  | Notes                                                                                     |
| -                | -                      | -                                                                                         |
| Protocol version | 3.1                    | Chosen for maximum compatibility with router software                                     |
| Signature type   | Ed25519-SHA512         | Default value; recommended by the I2P project                                             |
| Encryption type  | MLKEM-768/ECIES-X25519 | Same reason as above; falls back to the second if the first is unavailable. Post-quantum. |
| Stream timeout   | 60 seconds             | Recommended by the I2P project                                                            |
| Tunnel quantity  | 3                      | Ensures consistent behavior between router implementations                                |

At a later date, when support arrives, the signature type parameters should be updated to use post-quantum schemes (if not already the default by then).

## Configuration

To use SAM functionality, the I2P router's SAM bridge must first be enabled. The bridge is enabled by default in [`i2pd`](https://github.com/PurpleI2P/i2pd) (C++), but not in [Java I2P](https://github.com/i2p/i2p.i2p).

In Java I2P, go to http://127.0.0.1:7657/configclients and start the SAM Application Bridge using the "Start" button. You may also want to enable the SAM Application Bridge by default by checking the "Run at Startup" box and clicking "Save Client Configuration."

If the SAM bridge is disabled in `i2pd`, ensure `sam.enabled=true` is set in the `i2pd.conf` file.

While default router settings will likely be sufficient for most users, the maximum bandwidth usage of the I2P router can be reduced if it is too high. See [here](https://github.com/bitcoin/bitcoin/blob/master/doc/i2p.md#bandwidth) for more information.

## Private key storage

The private destination key is automatically generated by the I2P router, and stored as `i2p_private_key` in the `bitmonero` data directory (by default). You will likely want to create backups of this key.
